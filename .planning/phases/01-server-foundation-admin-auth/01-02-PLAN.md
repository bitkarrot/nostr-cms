# Plan 01-02 — NIP-98 verification + master-pubkey check + admin endpoint scaffold

<objective>
Implement server-side NIP-98 verification using `@nostrify/nostrify`'s `NIP98.verify` (pure crypto via nostr-tools `verifyEvent`), reconstruct the public request URL from `X-Forwarded-*` headers so verification succeeds behind nginx (T-01-03, the #1 bug risk), resolve the site master pubkey from `MASTER_PUBKEY` env or `/.well-known/nostr.json` fetch (fail-closed), and wire a Hono `nip98Auth` middleware + scaffold admin endpoint that rejects unsigned/non-master requests (401/403) and accepts valid NIP-98 from the master pubkey. Covers SRV-03 and the SRV-01 health-endpoint acceptance (health route created in 01-01; this plan fills its test and adds the admin seam). Implements D-04 (health public, no auth).
</objective>

## Artifacts this phase produces

- `server/auth/nip98.ts` — `publicRequest(c: Context): Request` (reconstructs the public URL from `X-Forwarded-Proto` / `X-Forwarded-Host` / `Host` + the request path) and `verifyNip98(request: Request): Promise<NostrEvent>` (thin glue over `NIP98.verify` from `@nostrify/nostrify`, re-throwing on failure).
- `server/auth/master-pubkey.ts` — `resolveMasterPubkey(): Promise<string>` reading `MASTER_PUBKEY` env first (lowercased, trimmed), else fetching `${SWARM_BASE_URL}/.well-known/nostr.json` and returning `names._` (lowercased, trimmed); returns `''` / throws on fetch failure so the middleware fails closed.
- `server/middleware/nip98Auth.ts` — `nip98Auth({ masterResolver })` Hono middleware: calls `verifyNip98(publicRequest(c))`, on throw returns 401; compares `event.pubkey` to the resolved master pubkey, mismatch returns 403; on success sets `c.set('pubkey', event.pubkey)` and calls `next()`.
- `server/routes/admin.ts` — scaffold admin route(s) under `/api/email/admin/*` (e.g. `GET /api/email/admin/ping` returning `{ ok: true, pubkey }`) mounted behind `nip98Auth`, proving the auth seam works end-to-end.
- `server/index.ts` (edit) — registers the admin routes + `nip98Auth` middleware after the public health route.
- `server/auth/nip98.test.ts`, `server/auth/master-pubkey.test.ts` — filled from the 01-01-00 stubs.
